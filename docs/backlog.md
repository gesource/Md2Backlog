# Backlog 記法リファレンス

## 見出し

`*`（アスタリスク）の個数がレベルを表す。

```
* 見出し1
** 見出し2
*** 見出し3
**** 見出し4
***** 見出し5
****** 見出し6
```

## インライン装飾

| 記法 | 表示 |
|------|------|
| `''text''` | **太字** |
| `'''text'''` | *斜体* |
| `'''''text'''''` | ***太字＋斜体*** |
| `%%text%%` | ~~打消し線~~ |
| `` `code` `` | インラインコード |

## リスト

### フラットなリスト

番号なしリストは `-`、番号付きリストは `+` を使う。

```
- item 1
- item 2
- item 3

+ first
+ second
+ third
```

### ネストリスト

**文字数 = ネストレベル、文字種 = リスト種別** で表現する。

```
- level 1 bullet
-- level 2 bullet
--- level 3 bullet

+ level 1 ordered
++ level 2 ordered
+++ level 3 ordered
```

レベルと種別は独立しており、親子間でリスト種別を自由に切り替えられる。

```
- a
++ a1
--- a1a
--- a2a
++ a2
--- a2a
- b
++ b1
--- b1b
++ b2
```

上記の Backlog 記法は以下のように表示される：

```
• a
  1. a1
      ■ a1a
      ■ a2a
  2. a2
      ■ a2a
• b
  1. b1
      ■ b1b
  2. b2
```

#### ネストのルール

- `-` が 1 個 → level 1 番号なし（•）
- `--` が 2 個 → level 2 番号なし（■）
- `---` が 3 個 → level 3 番号なし（■）
- `+` が 1 個 → level 1 番号付き（1.）
- `++` が 2 個 → level 2 番号付き（1.）
- `+++` が 3 個 → level 3 番号付き（1.）

#### Markdown との対応

| Markdown | Backlog |
|----------|---------|
| `- item` | `- item` |
| `  - nested` | `-- nested` |
| `    - deep` | `--- deep` |
| `1. item` | `+ item` |
| `   1. nested` | `++ nested` |
| `- bullet`<br>`  1. ordered sub` | `- bullet`<br>`++ ordered sub` |

## 制約・非サポート事項

### リストアイテム内のコードブロック

Backlog 記法ではコードブロック（`{code}...{/code}`）はリストアイテムの子要素として扱われない。
コードブロックは常に独立したブロック要素になり、リストを分断する。

```
+ aaa

{code}
code
{/code}

+ bbb
```

上記は以下のように表示される（リストが分断され、番号がリセットされる）：

```
1. aaa

[code block]

1. bbb   ← 1 から再スタート
```

Markdown でリストアイテム内にコードブロックを書いた場合、Backlog 変換時に意図通りの表示にならない点に注意。

## 引用

ブロック引用には `{quote}...{/quote}` を使う。内部では Backlog 記法が有効なため、リストやコードブロックなど複数のブロック要素を含められる。

```
{quote}
''重要なメモ:''
- item 1
- item 2
{/quote}
```

`> ` プレフィックス記法も存在するが、複数行・複数要素を含む場合は `{quote}...{/quote}` を使うこと。
